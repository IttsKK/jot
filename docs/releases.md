# Jot Release Setup

This repo uses GitHub Actions for:

- CI on `main` (`.github/workflows/ci-main.yml`)
- Tagged release build + Sparkle appcast publish (`.github/workflows/release.yml`)

## One-time GitHub setup

1. Add repository secret:
- Name: `SPARKLE_PRIVATE_KEY`
- Value: private EdDSA key generated by Sparkle `generate_keys` (the private key, not `SUPublicEDKey`)

2. Enable GitHub Pages:
- Repository Settings -> Pages
- Source: `GitHub Actions`

3. Confirm feed URL in app:
- `Jot/App/Info.plist` should contain:
- `SUFeedURL = https://ittskk.github.io/jot/appcast.xml`

## Day-to-day flow

1. Merge into `main`
- CI runs automatically (build + tests)

2. Ship a release
```bash
git tag v1.0.1
git push origin v1.0.1
```

The release workflow will:

- build `Jot.app`
- package `Jot-<version>.zip` and `Jot-<version>.dmg`
- generate signed `appcast.xml`
- create/update GitHub Release `v<version>` with assets
- publish `appcast.xml` to GitHub Pages

## Local manual release (optional)

You can still run local packaging:

```bash
./scripts/release.sh --version 1.0.1 --owner IttsKK --repo jot
```

For local appcast generation, either:

- use local keychain account (`SPARKLE_KEY_ACCOUNT`), or
- pass a file key (`SPARKLE_ED_KEY_FILE`), or
- pass key directly (`SPARKLE_PRIVATE_KEY`)
